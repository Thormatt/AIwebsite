Data Readiness Scorecard

AI Enablement Assessment

Meridian Financial Services

February 2026 | Confidential

58/100
Moderate Data Readiness

"You have data, but it's siloed and poorly governed"

Data Dimension Scores

Data Availability
75/100

Good Comprehensive customer, transaction, and operational data across core banking, CRM, and wealth management systems.

Data Quality
45/100

Needs Work Inconsistent standards, duplicate records in CRM (est. 12%), missing/incomplete fields in 20% of wealth management profiles.

Data Governance
30/100

Critical Gap No data catalog, no metadata management, unclear data ownership, ad hoc quality checks.

Data Security & Privacy
65/100

Acceptable SOC 2 compliant, encryption at rest/transit, role-based access controls, but no AI-specific privacy frameworks.

Data Accessibility
50/100

Moderate APIs exist but not well-documented, cross-functional data requests take 2-4 weeks, siloed access permissions.

Data Lineage & Observability
40/100

Weak Limited traceability, no automated lineage tracking, manual impact analysis when schema changes.

Top 3 AI Blockers

1. No Data Catalog or Metadata Management

Teams don't know what data exists, where it lives, or who owns it. Data scientists spend 60% of their time hunting for data instead of building models.

Impact: Delays every AI initiative by 4-6 weeks

2. Inconsistent Quality Standards Across Systems

Each department defines "customer" differently. Core Banking uses SSN as key, CRM uses email, Wealth Management uses account ID. No master data management (MDM) strategy.

Impact: AI models trained on bad data = bad predictions

3. Siloed Access (Marketing Can't See Sales Data)

Rigid access controls prevent cross-functional AI use cases. Marketing team cannot access transaction data needed for personalization AI, citing "privacy concerns" without formal policy.

Impact: Blocks 3 of 4 prioritized AI use cases

Quick Wins (30-60 Days)

Deploy Data Catalog (e.g., Alation, Collibra)

Automated discovery and cataloging of all data assets. Teams can search, understand, and request access to data in self-service mode.

Effort
Low
Impact
High

Implement Basic Data Quality Rules

Define and enforce 5-10 critical quality rules (e.g., required fields, format validation, duplicate detection) using existing tools (dbt, Great Expectations).

Effort

Impact

Legal + Compliance + IT define formal policy for cross-departmental data sharing with PII masking and audit logging.
